Records & Lists in Power Query (M Language)

An In-Depth Guide

What Are Records and Lists, Really?

The M Language Data Model

Power Query (M language) has a type system built around a few core value types. Two of the most fundamental are:

Type Symbol Nature
📦List { } Ordered, unnamed
🧾Record [ ] Unordered, named

The brackets are not just syntax — they are the identity of each type.

Lists — What Are They Really?

A list is a sequence of values where position is everything.

{10, 20, 30}

Key characteristics:

  • The values have no names — only index positions (0-based)
  • The order matters — {10, 20, 30} ≠ {30, 10, 20}
  • The items don't have to be the same type
{1, "hello", true, {1,2}, [Name="Ahmad"]}

Think of a list like a numbered shelf — you retrieve things by their slot number. 

myList = {10, 20, 30}

myList{0}   // → 10
myList{2}   // → 30
A list answers the question: “What is at position N?”

Records — What Are They Really?

A record is a collection of named fields, where the name is everything.

[Name = "Ahmad", Age = 30, City = "Cairo"]

Key characteristics:

  • Every value has a field name
  • Order does not matter
  • You access values by name, not position
myRecord = [Name = "Ahmad", Age = 30]

myRecord[Name]   // → "Ahmad"
myRecord[Age]    // → 30
A record answers the question: “What is the value of field X?”

The Core Difference — A Mental Model

List Record
Container 📊 Numbered slots 🏷️ Labeled boxes
Access by Position {0} Name [FieldName]
Order matters? ✔ Yes ❌ No
Has names? ❌ No ✔ Yes
Heterogeneous? ✔ Yes ✔ Yes
Duplicate keys? N/A ❌ Not allowed

How They Sit Inside the Bigger Picture

Table

├── Row 1 → [Name="Ahmad", Age=30, City="Cairo"]
├── Row 2 → [Name="Sara", Age=25, City="Alex"]

├── Column "Name" → {"Ahmad", "Sara"}
└── Column "Age" → {30, 25}

This is why understanding lists and records is not optional — tables are literally built from them.

A List of Records = A Table

#table(
    {"Name", "Age"},
    {
        {"Ahmad", 30},
        {"Sara", 25}
    }
)

Internally, each row becomes: [Name="Ahmad", Age=30]

MyTable{0} → returns a record
MyTable[Name] → returns a list
